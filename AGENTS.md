# AI Agents & Tools

ばんえい競馬予測AIモデル開発に関連するエージェントやツールの役割、構成などをまとめるファイルです。

## プロジェクト遵守事項

1. **データの非公開**: 今後スクレイピングしたデータはGitHubで公開せず、追跡対象外（`.gitignore`）のディレクトリに読み込んでから使用する（スクレイピングコードの公開は可）。
2. **指示ベースの進行**: プロジェクトはユーザーのペースで進行する。未指示の作業を勝手に進めたり、先の工程を推測して提案するようなおせっかいは厳禁。
3. **作業前の事前通知**: ファイルの書き込み等の作業を開始する前に、必ず「何をしようとしているか」を文章で通知する。事後報告のみの通知は禁止。
4. **コミットの提案**: リポジトリはGit管理のため、AI自身がコミットする必要はないが、適切なタイミングでのコミット提案とコミットメッセージの例文提示は行う（実際のコミット操作はユーザーが行う）。
5. **簡潔な回答**: AIトークンの浪費を防ぐため、相槌や枕詞的な文章は使用しない。

## プロジェクト概要
帯広競馬場で開催される「ばんえい競馬」に特化した高精度着順予測AIモデルの開発プロジェクト。競馬プロ予想家（netkeiba等）の予測精度を凌駕し、期待回収率をプラスに持っていくための「ガチ性能」を目指す。

## 現在のモデル構成と実績精度

### 1. 特徴量システム（計54次元）
- **馬場水分物理相互作用**: `power_moisture_interaction` (馬の負担重量比 × 水分量)、`sled_weight_moisture_interaction` (ソリ重量 × 水分量)。水分によるソリの滑り摩擦の非線形変化を反映。
- **気象庁外部データ**: `data/raw/weather/jma_obihiro_weather.csv` (降水量、気温、湿度、風速、日照時間、降雪量、積雪量)。
- **調子・疲労・勝負駆け**: `jockey_upgrade_factor` (鞍上強化の度合い)、`fatigue_index` (前走馬体重減 × レース間隔)、`recent_form_score` (近走3走の加重順位平均)。

### 2. 予測アーキテクチャ
- 1着・2着・3着それぞれの予測に特化した LightGBM（`CalibratedClassifierCV` による勝率キャリブレーション適用）モデル。
- レースごとに、被りのない順列の確率積（$P_1 \times P_2 \times P_3$）を最大化する「最適着順割り当てアルゴリズム」による3連単フォーメーション生成。

### 3. 実績精度（2026年検証用データ 895レース）
- **1着 ドンピシャ的中率**: **24.80 %**
- **2着 ドンピシャ的中率**: **17.99 %**
- **3着 ドンピシャ的中率**: **15.31 %**
- **馬単 完全ピタリ (1点)**: **5.81 %**
- **3連単 ズバリ (1点)**: **1.34 %**
- **★ 厳選・勝負レース（AI確信度40%以上、1日約1レース出現）的的中率**: **★ 47.4 % ★** (単勝ほぼ2回に1回的中)
- **★ 厳選・勝負レース（AI確信度50%以上）的中率**: **★ 53.8 % ★**

## プロジェクトの主要スクリプト

| スクリプトパス | 役割 |
| :--- | :--- |
| `src/data/scrape_jma_weather.py` | 気象庁から帯広の過去詳細気象データを取得しCSV化。 |
| `src/features/build_features.py` | 54次元特徴量（水分相互作用、気象マージ、調子、疲労）の生成。 |
| `src/models/train_position_models.py` | 1〜3着個別予測モデルの学習と最適着順割り当てシミュレーション。 |
| `src/models/evaluate_confidence.py` | AI確信度の閾値による「勝負レース厳選」時の的中率検証。 |
| `src/models/predict_netkeiba_style.py` | 最新の出走表から「AIガチ度（本命信頼度）」判定付きプロ予想紙出力を生成。 |

## 次回作業への引き継ぎ事項

1. **特徴量システムのさらなる拡張**:
   - 馬ごとの馬場水分適性のビニング（例: 重馬場専用、良馬場専用）を考慮した、`track_specialist_factor` のベイズ平滑化など。
2. **ハイパーパラメータの最適化 (Optuna)**:
   - 特徴量が54次元へと大幅に拡張されたため、LightGBMモデルの `n_estimators`, `num_leaves`, `learning_rate`, `min_child_samples` 等のハイパーパラメータ再チューニングが非常に有効。
3. **回収率（ROI）志向の投票ロジック実装**:
   - 的中確率だけでなく、オッズデータ（`odds_num`）を用いて期待値（的中確率 × オッズ）がプラスの組み合わせのみに投票するシミュレータの実装。
