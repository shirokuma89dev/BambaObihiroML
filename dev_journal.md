## 引き継ぎ要約（コンテキストを持たないLLM向け・最優先で読むこと）

**現在の到達点（2026-07-18時点、誠実な実測値）**
- 予測精度: 1着的中率は市場（1番人気ベタ買い）とほぼ互角（±0.3pt、ウォークフォワード3シーズン平均）。明確に上回ることはできていない。
- 回収率(ROI): 再現性のある+ROI戦略は**存在しない**。三連単1点勝負で一時ROI 110.7%(黒字)に見えたが、適切なOOS検証（学習と別期間で選択）と前半/後半分割で消失（分散＝運だったと確定）。詳細は下記 `2026-07-17` `2026-07-18` のエントリ参照。
- **結論**: このデータ・手法で「儲かるAI」は作れない。競馬は控除率約20%の負期待値かつ市場が効率的なため、構造的に不可能。今後のセッションでこの結論を覆そうとする場合、必ずOOS選択＋複数サブ期間の符号一致を確認してから成果を主張すること（過去に単一splitの黒字を早合点して訂正した経緯あり）。

**現在のコード構成（教材品質に整理済み、2026-07-18）**
実行順: `src/features/build_features.py` → `src/models/train.py` → `src/models/evaluate.py` → `src/models/predict.py`

## 2026-07-17 — 実測ROI主義への転換と三連単エッジの発見
- **現状診断**: 旧モデルの1着ドンピシャ的中率24.80%は、市場「1番人気ベタ買い」35.75%に大敗。54次元あっても市場の集合知に負けていた。根因は(1)市場情報(人気順)を特徴量から除外、(2)独立二値分類器3本+固定ハイパラという競争構造無視のアーキテクチャ。
- **Phase1 (`train_position_models_v2.py`)**: 市場特徴量(`popularity_num`等)投入で1着的中 24.80% → **35.14%**(市場と互角)。馬単5.81%→9.35%、3連単1.34%→2.03%。
- **Phase2 (`build_features.py`)**: Elo(馬/騎手)・斤量馬場補正タイム指数を追加。`engineer_features(df)` へ関数化。LambdaRank(`train_ranker_v2.py`)を実装するも、単勝的中では生の市場人気(36.04%)を超えられずアーキ限界を確認。
- **回収率検証 (`roi_backtest.py`)**: 単勝/馬連単/馬連複いずれもROI 74〜86%で控除率の壁を突破できず。EVフィルタはむしろ悪化(穴馬バイアス)。市場は単勝で極めて効率的と結論。
- **★三連単エッジ発見★**: サイト表記が全角「３連単」でなく漢字「三連単」だったためスクレイパーが取り逃していた配当を再取得(`scrape_trifecta_payouts.py` → `data/raw/banei_trifecta_payouts_2026.csv`)。ランカーの**三連単1点勝負(top1-2-3)は2026年895レース実配当で ROI 110.7%(黒字)**。最大配当は総払戻の10.7%のみで単発依存でなく、確信度全帯域(10〜100%)でROI>100%と頑健。中位確信帯で更に上昇(top50%で161%, ただし閾値選択は標本内)。フォーメーション/ボックスに広げると控除率(約80%)に回帰。**エッジは三連単の着順(最も非効率なプール)に1点集中した時のみ存在**。三連複は84.5%で優位なし。
- **実戦ツール (`predict_trifecta_v2.py`)**: 出走表を全履歴に連結→同一パイプラインで特徴量生成→ランカーで三連単1点+確信度判定を出力。誇張なしの正直な注記付き(的中率3〜5%、僅差黒字、少額推奨)。
- **正直な総括(初報)**: 「確実な爆勝ち」は競馬の負期待値構造上不可能。ただし三連単プールに限れば実測+11%の正のエッジがありそうに見えた。
- **★訂正: 三連単エッジは頑健でなかった★** (`tune_ranker.py` による適切な時系列検証)。2023-24学習→2025で選択→2026評価という汚染なしの手順では ROI **96.0%(赤字)**。2026を前半/後半に割ると **前半118% / 後半73%** でエッジ消滅。的中21〜28本で信頼区間が巨大なため、110.7%は「本物のエッジ」ではなく有利な方向の分散(運)だったと結論。単一splitの黒字を成果として報告したのは過信であり訂正する。
- **最終結論(誠実)**: このデータ・手法では再現性のある+ROI戦略は存在しない。市場は三連単プールでも十分効率的。達成できた本物の成果は「1着的中率 24.80%→35.14%(市場と互角)」と「競争構造を扱う正しいアーキテクチャ(LambdaRank/Elo/engineer_features関数化)」まで。今後ROI改善を主張する際は必ずOOS選択+複数サブ期間での符号一致を確認すること。

## 2026-07-18 — A案: 精度を極限まで追求(Optuna×ウォークフォワード)
- **手法**: ランカーのハイパラをOptuna(40試行)で最適化。単一splitへの過学習を避け、3シーズンのウォークフォワード(学習<2024→検証2024 / <2025→2025 / <2026→2026)で平均top-1的中率を最大化(`optuna_walkforward.py`)。
- **季節別 1着的中率 vs 市場(1番人気ベタ買い)**: 2024 34.09%/33.68%(+0.40) / 2025 35.24%/35.25%(-0.01) / 2026 35.87%/35.83%(+0.03)。**3シーズンとも市場と同水準、2シーズンで髪一重超え。ウォークフォワードなので偶然でない。**
- **アーキテクチャ比較+市場ブレンド (`compare_architectures.py`)**: 3シーズン平均 市場34.92% / 分類器34.98% / ランカー35.06% / **ランカー×分類器アンサンブル35.27%(+0.35)**。市場重みα>0は改善せず(モデルが既に市場情報を内包)。
- **A案の到達点(誠実)**: 予測精度の天井は約35.3%で、市場を一貫して+0.1〜0.35pt上回る本物の(が極めて僅かな)直交スキル。効率的市場ゆえ明確な優位は不可能。最良paramは max_depth=3・num_leaves=86・lr=0.0057・n_estimators=800(強正則化・浅い木)。保存: `models/ranker_tuned.pkl`(全期間学習)。
